# 阿尔比恩在线 数据查询接口文档（欧服 / Europe·Amsterdam）

> 默认面向**欧服**，域名一律用欧服变体。换区只改域名段，路径/字段一致。
> 标注：✅ = 本机 curl 实测确认（2026-06-14） ｜ 📄 = 社区资料/逆向（未逐一实测）
> 阿尔比恩官方**没有**对外 API 文档，以下接口均为社区逆向、被官方默许，无 SLA，会偶发变慢/限流。

---

## 一、生态总览（先看这张图）

```
                         ┌─ 玩家/公会/联盟/击杀/战役/声望（实时，仅快照+最近N条，无历史）
官方 Gameinfo API ───────┤
  gameinfo-ams...        └─ kill event 里带死者完整装备(10槽)+背包(数组)

AODP 市场 API ───────────── 物品价/金价/历史（玩家众包上传，低频物品仍可能稀疏）

静态资源 ─────────────────── 图标渲染(render) + 物品字典(ao-bin-dumps，含中文名)

第三方站(albionbb/murderledger/albiondb)
  = 上面官方数据的「聚合 + 索引 + 历史存档」层
    它们后端都在定时轮询官方 Gameinfo，存进自己的库再对外服务
```

**一句话**：所有数据的源头只有两个——官方 **Gameinfo**（战斗/玩家）和 **AODP**（市场）。第三方站只是帮你做好了聚合和历史。做功能时，要么直连源头自己聚合（可控、无反爬），要么蹭第三方现成接口（省事，但部分有 Cloudflare 拦截）。

## 二、官方 Gameinfo API（核心数据源）

- **Base（欧服）**：`https://gameinfo-ams.albiononline.com/api/gameinfo` ✅
- 其他区：美服 `gameinfo.albiononline.com`、亚服 `gameinfo-sgp.albiononline.com`
- 无鉴权、返回 JSON、分页用 `limit`/`offset`、时间窗用 `range`
- ⚠️ 玩家/公会要先拿到 **ID**（base64 串），用 `/search` 换取

### 2.2 击杀事件结构 ✅（装备/背包价值的数据基础）

`/events` 或 `/events/{id}` 每条事件实测字段树：

```
{
  EventId, TimeStamp, TotalVictimKillFame, BattleId, Type:"KILL",
  numberOfParticipants, groupMemberCount, Location, KillArea,
  Killer  { ...同 Victim 结构 },
  Victim  {
    Id, Name, GuildName, GuildId, AllianceName, AllianceId, AllianceTag,
    AverageItemPower,           // 平均装备档次(IP)
    DeathFame, KillFame, FameRatio,
    LifetimeStatistics { ... }, // 见 2.3
    Equipment {                 // 死者身上 10 个槽
      MainHand, OffHand, Head, Armor, Shoes, Bag, Cape, Mount, Potion, Food
      // 每个槽: { Type, Count, Quality, ActiveSpells, PassiveSpells, LegendarySoul } 或 null
    },
    Inventory [ {Type,Count,Quality,...}, ... ]  // 背包数组(实测可达数十件)
  },
  Participants[], GroupMembers[]   // 参战者/小队成员(用于算助攻)
}
```

要点（实测）：
- `Type` 形如 `T4_2H_DUALSWORD@1`，`@1` 是附魔等级（0~4，无后缀=0）。
- `Quality` 是数字：0/1=普通, 2=精良, 3=杰出, 4=卓越, 5=大师。
- 背包里有不可交易物品（如 `T4_SKILLBOOK_NONTRADABLE`），计价时查不到价，要跳过。

### 2.3 玩家终身声望（PvE/采集/制作）✅

LifetimeStatistics {

## 三、AODP 市场 API（物价 / 金价）

- **Base（欧服）**：`https://europe.albion-online-data.com` ✅
- 其他区：美服 `west.`、亚服 `east.`
- 限流 **180 次/分**，URL 上限 4096 字符；item 可逗号批量
- ⚠️ **市场数据可能稀疏**：很多城市/质量返回 0（=无挂单），计价要做兜底

| 端点 | 参数 | 返回(实测字段) |
|---|---|---|
| `/api/v2/stats/prices/{items}.json` ✅ | locations, qualities | `[{item_id, city, quality, sell_price_min, sell_price_min_date, sell_price_max, buy_price_min, buy_price_max, ...}]` |
| `/api/v2/stats/history/{items}.json` ✅ | locations, qualities, time-scale(1/6/24) | `[{location, item_id, quality, data:[{item_count, avg_price, timestamp}]}]` |
| `/api/v2/stats/gold.json` ✅ | count 或 date&end_date | `[{price, timestamp}]` |

要点（实测）：
- **支持附魔后缀**：`/prices/T5_ARMOR_PLATE_SET1@1.json` 合法（但冷门词条常无挂单返回 0）。
- 价格有**离群噪音**：实测某城 `T4_BAG` 挂单 333333（恶意挂单），计价别直接取单点，用多城/历史均价/中位数。
- `sell_price_min` = 当前最低卖价；`buy_price_max` = 当前最高收购价；日期 `0001-01-01` 表示无数据。

## 四、静态资源

### 4.1 物品图标 render 📄
`https://render.albiononline.com/v1/item/{Type}.png?quality={0-5}&size={1-217}`
- `Type` 可带附魔，如 `T4_OFF_SHIELD@1`。例：`https://render.albiononline.com/v1/item/T4_2H_DUALSWORD@1.png?quality=2`

### 4.2 物品字典 ao-bin-dumps 📄
- 仓库：`github.com/ao-data/ao-bin-dumps`，文件 `formatted/items.json`、`items.txt`
- 字段：`UniqueName`(=事件里的 Type 基名) ↔ `Index`(内部ID) ↔ `LocalizedNames`(多语言，含中文)
- 用途：把 `T4_2H_DUALSWORD` 翻译成「双剑」，做物品名/分类映射

## 五、三个第三方站：数据来源实测

### 5.1 albionbb（ZvZ 战报）✅ —— 可服务端直连，无 Cloudflare

前端 `europe.albionbb.com` 调用自建后端：

| 端点 | 参数 | 返回 |
|---|---|---|
| `https://api.albionbb.com/eu/battles` ✅ | search, minPlayers, page | `[{albionId, startedAt, totalFame, totalKills, totalPlayers, alliances:[{name,killFame}], guilds:[{name,alliance,killFame}]}]` |
| `https://api.albionbb.com/eu/battles/kills?ids=ID1,ID2` ✅ | ids(可多个合并) | `[{EventId, TimeStamp, TotalVictimKillFame, Killer{...}, Victim{...}}]` |

- region 段 `eu`（对应美服/亚服换段）。
- kills 是**裁剪版**：每个击杀只保留 `MainHand`+`Mount`（Name/Type/Quality），不含其他装备槽和背包。
- kills 响应 `Cache-Control: max-age=604800 immutable`（缓存 7 天）。
- ✅ **curl 无 referer/origin 也能直连**（CORS 头只限制浏览器跨域，不挡服务端）→ 做 ZvZ 功能可直接用，最省事。

### 5.2 murderledger（PvP 玩家维度）⚠️ Cloudflare 全站拦截

- 站点按区服分域，**Cloudflare Turnstile 覆盖全站含 `/api`**，本机 curl 实测返回 403「Just a moment」，自动化难以程序调用（你用真实浏览器访问没问题）。
- 数据来源 📄：定时轮询官方 `/api/gameinfo/events` → 存 PostgreSQL → 按玩家维度建索引（击杀/死亡历史、武器对战矩阵）。
- 前端路由 📄：`/api/players/{name}/events`、`/api/players/{name}/stats/weapons`、`/api/player-search/{name}`、`/weapon-matrix`。
- 开源：`gitlab.com/albion-murder-ledger`，镜像 `github.com/faervan/albion-killarchive`。
- **结论**：它没有官方 events 给不了的独家数据，只是做了玩家维度索引。要程序化用，直接拿官方 `/events` 自己按玩家聚合即可，绕开 CF。

### 5.3 albiondb（个人/公会/联盟 + 历史）⚠️ Cloudflare 拦截

- 站点按区服分域，**Cloudflare JS challenge**，自动化过不去（本机实测卡在验证页）。
- 数据来源 📄：定时轮询对应区服 Gameinfo → 存自建库，**主打官方 API 给不了的历史**（公会历史、联盟成员变迁、长期追踪）。
- 前端路由 📄：`/player/{name}`、`/guild/{name}`、`/alliance/{name}`。
- 无公开源码。
- **结论**：独家价值是「历史存档」。若你的功能需要历史，要么自己定时抓官方 `/players` `/guilds` 存库，要么人工查它的网页。

| 你要的 | 怎么取 | 推荐 |

### 七、装备 / 背包价值 计算机制（实测可行）

没有「价值」这个现成字段，要自己组合两个源：

```ts
// 伪代码：估算一次击杀里死者的装备+背包价值（欧服）
const SLOTS = ["MainHand","OffHand","Head","Armor","Shoes","Bag","Cape","Mount","Potion","Food"];

async function estimateValue(event) {
  const v = event.Victim;
  const items = [
    ...SLOTS.map(s => v.Equipment[s]).filter(Boolean),
    ...(v.Inventory ?? []).filter(Boolean),
  ];
  let total = 0;
  for (const it of items) {
    // it.Type 形如 T5_ARMOR_PLATE_SET1@1；按 it.Quality 查对应质量
    const price = await aodpPrice(it.Type, it.Quality); // GET europe.albion-online-data.com/.../prices/{Type}.json?qualities={Quality}
    if (price > 0) total += price * (it.Count ?? 1);     // 查不到价(不可交易/无挂单)记 0
  }
  return total; // 单位：银币 silver
}
```

注意（都来自实测）：
1. `Type` 带附魔后缀，AODP 直接拿带 `@` 的去查即可。
2. 用 `it.Quality` 对齐 AODP 的 `qualities` 参数。
3. 市场挂单稀疏 + 有离群噪音 → 建议查**多城取中位/最低正价**，或用 `/history` 近 N 天 `avg_price` 更稳。
4. 不可交易物（技能书等）查不到价，跳过即可。
5. 批量优化：一次击杀几十件物品，用逗号批量查 + 本地缓存价格，别逐件打 AODP（180/分限流）。

## 八、做功能的两条路线 + 注意点

**路线 A（推荐）直连官方源**：`gameinfo-ams` + `europe.albion-online-data.com`。完全可控、无 Cloudflare、字段最全（含完整装备+背包）。代价是要自己做聚合/历史存档。

**路线 B 蹭第三方**：ZvZ 直接用 `api.albionbb.com/eu/*`（无 CF、可直连，省最多事）；murderledger / albiondb 有 Cloudflare，不适合程序调用，只适合人工查或当灵感参考。

**通用注意点**：
- **区服一致**：战斗用 `gameinfo-ams`、市场用 `europe.albion-online-data.com`、ZvZ 用 `api.albionbb.com/eu`，三者都得是欧服，别混区。
- **先 ID 后查**：玩家/公会先 `/search` 拿 base64 ID。
- **限流与缓存**：AODP 180/分；官方无明确限流但会变慢，自己加缓存 + 退避。
- **数据时效**：官方 `/events` 只给最近一段；要历史必须自己落库。
- **稳定性**：官方 API 偶发数天故障（社区常态），功能要容错。

## 九、能复现这三个站的什么功能

三站数据源都是官方 gameinfo 的**实时流 + 当前快照**，所以判断能否复现，关键看功能依不依赖「历史」。分三档：

- 🟢 **拉了就能做**：官方接口直接给，无需积累
- 🟡 **得先跑采集器攒数据**：官方只给「现在」，功能靠历史聚合 → 必须自己持续落库，攒够时间才有内容
- 🔴 **做不了**：数据不在游戏 API 内

### albionbb（ZvZ）— 实测，几乎全能即时复现

| 功能 | 接口 | 档 |
|---|---|---|
| 最近 ZvZ 战役列表 | `/battles` 或 `api.albionbb.com/eu/battles` | 🟢 |
| 战役详情（击杀+装备） | `/events/battle/{id}` 或 `/eu/battles/kills?ids=` | 🟢 |
| 多场合并战报 | `ids=` 传多个再自己聚合 | 🟢 |
| 按公会/玩家过滤 | `?guildId=` / 客户端筛 | 🟢 |

> 要完整装备(10槽+背包)用官方 events；albionbb 的 kills 只给主手+坐骑。

### murderledger（PvP）— 招牌功能几乎都要自建库

| 功能 | 接口 / 做法 | 档 |
|---|---|---|
| 玩家最近击杀/死亡 | `/players/{id}/kills`·`/deaths`（仅最近约10条） | 🟢 |
| 玩家完整 ledger | 持续抓 `/events` 落库、按玩家索引 | 🟡 |
| 武器对战矩阵(1v1胜率) | 抓 events → 自己判 1v1 → 算胜率 | 🟡 |
| Build/套装胜率统计 | 同上，按装备组合聚合 | 🟡 |
| 终身 KD / 平均IP | `/players/{id}` 直接给 | 🟢 |
| 近N场武器分布 | 需历史库聚合 | 🟡 |
| 1v1 赛季榜 / MMR | 自定义算法 + 历史库 | 🟡 |
| VOD 视频集锦 | Twitch 录像，不在游戏 API | 🔴 |

> 关键：murderledger 的「1v1/2v2」不是官方标签，是它从 `Participants` 自己判定的；所有胜率/矩阵都靠历史 events 聚合，官方没有现成接口。

### albiondb（个人/公会/联盟）— 查询即时，历史要自建库

| 功能 | 接口 | 档 |
|---|---|---|
| 玩家档案(当前公会/声望/KD) | `/search` → `/players/{id}` | 🟢 |
| 玩家最近击杀/死亡 | `/players/{id}/kills`·`/deaths` | 🟢 |
| 公会成员 / 概览 | `/guilds/{id}/members`·`/data` | 🟢 |
| 联盟下属公会 | `/alliances/{id}` | 🟢 |
| 官方榜(击杀/PvE/采集声望) | `/events/playerfame`、`/players/statistics` | 🟢 |
| **玩家公会历史**（招牌） | 官方只给当前公会 → 每日快照 diff | 🟡 |
| 声望 / KD 趋势图 | 每日记录累计值 | 🟡 |
| 历史成员表 | 每日快照公会成员 diff | 🟡 |

> 关键：albiondb 的护城河就是「历史」——官方 API 是现在时，公会变迁/趋势都得自己按天攒。

### 一句话结论
- 🟢 **拉了就能做**：ZvZ 战报全套、玩家/公会/联盟当前档案、最近10条战斗、官方现成榜单、装备/背包估值。
- 🟡 **得先跑采集器攒数据**：一切「历史/趋势/胜率/矩阵/公会变迁」——这是三个站真正的壁垒，时间是攒出来的，不是接口给的。
- 🔴 **接口给不了**：murderledger 的 Twitch VOD。
