# 阿尔比恩 KOOK 机器人使用说明书

本文面向服务器管理员、补装审核员和普通成员，说明机器人怎么配置、怎么使用、怎么排错。

项目 GitHub 仓库：<https://github.com/Fat-Jan/Albion.git>

当前项目版本：`1.0`。机器人在线时，`/ping` 会返回 `pong v1.0`。

## 一、管理员首次配置

### 1. 邀请和权限

把机器人加入 KOOK 服务器后，确认机器人身份组具备这些权限：

- 管理身份组：用于发放会员身份组、补装审核身份组。
- 修改他人昵称：用于绑定通过后把 KOOK 昵称改成 Albion 角色名，或 `角色名 - 自定义昵称`。
- 读取和发送消息：用于接收指令、发送卡片。
- 在目标频道发卡片和按钮消息。

注意：KOOK 身份组有排序限制。机器人自己的身份组必须高于它要发放或撤销的身份组，否则会出现发组失败或改昵称失败。

### 2. 绑定 Albion 公会

管理员在任意可用频道输入：

```text
/绑定公会 <公会名>
```

机器人会返回候选公会卡片。管理员点击正确公会后，本 KOOK 服务器就会绑定到该 Albion 公会。

### 3. 配置基础频道和身份组

建议按顺序配置：

```text
/设置 会员身份组 @会员组
/设置 审批频道 #绑定审批频道
/设置 播报频道 #统一播报频道
/设置 击杀播报频道 #击杀播报频道
/设置 阵亡播报频道 #阵亡播报频道
/设置 战报推送频道 #战报推送频道
/设置 战报本会最小人数 20
/设置 成员变动频道 #成员变动频道
/设置 补装审核身份组 @补装审核员
/设置 补装初始化频道
/设置 大额阈值 100000
```

说明：

- `会员身份组`：玩家绑定通过后自动发放。
- `审批频道`：玩家绑定审批、补装审核身份申请会发到这里。
- `播报频道`：旧版统一播报频道；击杀/阵亡频道未单独配置时会兜底发到这里。
- `击杀播报频道`：我方击杀播报会发到这里。
- `阵亡播报频道`：我方阵亡播报会发到这里，也兼容 `/设置 死亡播报频道 #频道`。
- `战报推送频道`：ZvZ 聚合战报专用频道，也兼容 `/设置 战报频道 #频道`。自动推送会优先发到该频道。
- `战报本会最小人数`：自动战报推送阈值，默认 20；官方战役详情中本会参战人数低于阈值时不推送。
- `成员变动频道`：退会复查通知会优先发到这里。
- `补装审核身份组`：持有这些身份组的人可以审批补装、查看补装队列、标记已发放。
- `补装初始化频道`：新建 `🛡️补装中心` 分组和四个频道，并自动写入配置。
- `大额阈值`：死亡播报高亮门槛，单位是 fame。

推荐补装频道结构：

```text
🛡️补装中心
├─ 📥补装申请      成员可见，可发 /补装 /补装状态
├─ 🔍补装审核      仅补装组/管理可见
├─ 💰补装发放      仅补装组/发放组/管理可见
└─ 📣补装通知      成员可见，只发完成通知
```

也可以手动创建频道后绑定：

```text
/设置 补装申请频道 #补装申请
/设置 补装审核频道 #补装审核
/设置 补装发放频道 #补装发放
/设置 补装通知频道 #补装通知
```

旧命令 `/设置 补装频道 #频道` 仍保留为单频道兜底，但旧频道内容多时不建议复用。成员变动频道未配置时，退会通知会兜底发到播报频道，再没有播报频道会尝试阵亡播报频道、击杀播报频道，最后才发到审批频道。

### 4. 可信身份组（可选）

如果服务器里已有可信成员身份组，可以配置快速绑定：

```text
/设置 可信身份组 @老成员 @军团成员
```

持有可信身份组的成员绑定 Albion 角色时，如果角色在当前绑定公会内，会直接通过，不再走管理员审批。

### 5. AI 辅助（可选）

在 `.env` 中配置：

```dotenv
AI_ENABLED=true
AI_BASE_URL=https://api.longcat.chat/openai
AI_MODEL=LongCat-2.0-Preview
AI_API_KEY=<OpenAI 兼容密钥，可选>
LONGCAT_API_KEY=<本机密钥>
AI_TIMEOUT_SEC=20
AI_MAX_OUTPUT_TOKENS=800
```

`AI_API_KEY` 和 `LONGCAT_API_KEY` 二选一即可；两者都配置时优先使用 `AI_API_KEY`。

AI 只做说明和只读查询，不会自动审批绑定、补装、发组、撤组、改金额或标记已发放。开启后，AI 会出现在 `/战报`、`/助手`、`/补装解释`、补装审核卡和自动 ZvZ 战报卡中。输出层会拦截危险动作声明，并对疑似 Token/API Key 文本脱敏。

## 二、普通成员使用

### 1. 绑定角色

成员输入：

```text
/绑定 <角色名> [自定义昵称]
```

例如：

```text
/绑定 BEISHENGS 北笙
/绑定 DT777 - dt
```

不填自定义昵称时，KOOK 昵称会同步为 Albion 角色名；填写后统一同步为 `角色名 - 自定义昵称`。

机器人会检查：

- 角色是否存在。
- 角色是否在本服务器绑定的 Albion 公会中。
- 该角色是否已被其他 KOOK 成员绑定。
- 成员是否已有绑定或待审批申请。

通过后会自动：

- 写入绑定关系；自定义昵称保存在绑定记录的 `custom_nickname` 列，不会新增第二条绑定行。
- 发放会员身份组。
- 尝试把 KOOK 昵称改成 Albion 角色名，或 `角色名 - 自定义昵称`。

如果没有可信身份组，申请会进入审批频道，由管理员点击「通过」或「拒绝」。

待审批卡会直接显示绑定申请号、待审批状态和目标 KOOK 昵称。管理员处理后，机器人会把绑定结果卡发到成员变动频道；未配置成员变动频道时，兜底发回审批频道。结果卡会显示绑定申请号、申请人、角色、目标 KOOK 昵称和当前状态，并会尽量把原审批卡原地更新为已通过或已拒绝。如果发身份组或改昵称失败，结果卡会带告警，方便管理员继续处理。

### 2. 解绑角色

成员输入：

```text
/解绑
```

机器人会删除绑定关系，并尝试撤销会员身份组。

### 3. 查询自己或别人

已绑定成员可以省略角色名：

```text
/战绩
/估值
```

也可以指定角色名：

```text
/战绩 Latano
/估值 Latano
```

其他查询：

```text
/物价 老手级双剑
/金价
/榜单 pvp
/榜单 pve
/战役
/战报
/助手 我的绑定状态
/助手 我的最近死亡
/助手 我的补装状态
/补装状态
/补装解释 <申请号>
```

## 三、补装流程

### 1. 申请补装

成员必须先完成 `/绑定`，然后输入：

```text
/补装
```

建议在 `📥补装申请` 频道使用。机器人会列出该角色最近死亡记录。每条记录有两个按钮：

- 「详情」：查看死亡详情、装备明细、补装金额和官方击杀板链接。
- 「选这个补装」：提交补装申请。

补装金额口径：

- 只计算穿戴装备。
- 背包物品不计入补装。
- 背包金额仍会在死亡详情、播报总损失里展示，方便了解完整损失。

### 2. 审批补装

管理员或持有补装审核身份组的人，可以在 `🔍补装审核` 频道处理申请。

补装申请卡片会明确显示：

- 申请号和当前状态。
- 申请人。
- Albion 角色。
- 死亡时间、IP、击杀方、击杀声望、参与人数和官方击杀板链接。
- 穿戴装备文字明细。
- 补装金额。
- 「只计算穿戴装备，背包不计入补装」说明。
- 开启 AI 时，会附带「AI 审核提示」，只解释补装口径、缺价或异常点，不给通过/拒绝结论。

点击「通过」时，机器人会重新按当前市场数据刷新补装金额，再把申请状态改为 `approved`（待发放），并把待发放卡片转到 `💰补装发放` 频道。若配置了 `📣补装通知`，机器人也会在通知频道 @ 申请人，发送包含申请号、待发放状态、死亡摘要、装备明细、补装金额和官方击杀板链接的通知卡。原审核卡会尽量原地更新为已通过，避免旧按钮继续误导。

点击「拒绝」时，需要选择快捷拒绝理由；需要自定义理由时使用：

```text
/补装 拒绝 #申请号 理由文本
```

申请状态会改为 `rejected`，并记录拒绝原因和处理时间。处理时间在 KOOK 中显示为北京时间。若配置了 `📣补装通知`，机器人会在通知频道 @ 申请人，发送包含申请号、已拒绝状态、拒绝原因、死亡摘要、装备明细、补装金额和官方击杀板链接的通知卡。原审核卡会尽量原地更新为已拒绝。

### 3. 标记已发放

通过后的申请会在 `💰补装发放` 频道生成待发放卡片，卡片同样会显示死亡摘要、装备明细和官方击杀板链接。线下发银币或物资后，审核员选择实际发放方式：

```text
等额银币已发放
原样装备已发放
等价值物品已发放
```

需要备注时也可以使用：

```text
/补装 发放 #申请号 银币|装备|物品 备注
```

状态会改为 `paid`，并记录发放人、发放时间和发放方式。发放时间在 KOOK 中显示为北京时间。若配置了 `📣补装通知`，机器人会在通知频道 @ 申请人，发送包含申请号、已发放状态、发放方式、死亡摘要、装备明细、补装金额和官方击杀板链接的通知卡。原发放卡会尽量原地更新为已发放。

### 4. 查看补装队列

管理员或补装审核身份组可用：

```text
/补装 待处理
/补装 待发放
/补装 列表
/补装状态
/助手 补装队列概况
```

含义：

- `待处理`：状态为 `pending`，等待审批。
- `待发放`：状态为 `approved`，已经通过但还没标记发放。
- `列表`：最近补装记录，包含已通过、已拒绝、已发放等状态。
- `/补装状态`：普通成员查看自己的最近补装记录。

### 5. 申请补装审核身份

如果负责补装的人不是服务器管理员，可以输入：

```text
/补装审核
```

管理员会在审批频道收到申请卡。点击通过后，机器人会自动发放已配置的补装审核身份组。之后该成员就可以处理补装，但不会因此获得 KOOK 管理员权限。

补装审核身份申请卡会显示申请号和待审批状态。申请通过或拒绝后，机器人会把结果卡发到成员变动频道；未配置时兜底审批频道。结果卡会显示申请号、申请人、当前状态和北京时间审核时间，并会尽量把原申请卡原地更新为已通过或已拒绝。

## 四、自动任务

### 死亡播报

机器人普通时段每 4 分钟轮询亚服全局死亡事件，20:00-00:30 每 90 秒轮询一次，筛出当前绑定公会的击杀和阵亡。

播报特点：

- 区分我方击杀和我方阵亡。
- 可分别配置 `/设置 击杀播报频道` 和 `/设置 阵亡播报频道`；未配置时使用 `/设置 播报频道` 兜底。
- 大额死亡按 `/设置 大额阈值` 高亮。
- 阵亡播报会显示装备估值、背包估值和总损失。
- 首轮启动只记录历史，不会刷屏发送旧事件。

### 退会复查

每天 04:00，机器人会拉取当前 Albion 公会成员列表，与 KOOK 绑定表比对。

如果发现绑定角色已退会，会：

- 撤销该 KOOK 用户的会员身份组。
- 删除玩家绑定。
- 在成员变动频道发通知；未配置时兜底到播报频道或审批频道。

### ZvZ 战报

ZvZ 聚合战报使用独立频道配置：

```text
/设置 战报推送频道 #战报推送频道
/设置 战报本会最小人数 20
```

自动战报推送会优先发送到该专属频道，不复用击杀、阵亡或统一播报频道。北京时间 14:30 到次日 05:00 每 15 分钟检查一次；只有配置了战报推送频道时才请求 AlbionBB；命中绑定公会且本会参战人数达到阈值后发送卡片，开启 AI 时会附带「AI 摘要」。发送成功后写入 SQLite 去重表，重启后也不会重复推同一场。具体部署是否已跑过真实自动窗口，以对应实例的日志和状态记录为准。

### 价格参考库刷新

机器人每 3 天刷新一次 T4-T8 主手、双手、副手的低价参考库。

手动刷新命令：

```bash
.venv/bin/python -m scripts.refresh_price_reference
```

## 五、估值口径

估值数据来自亚服 AODP：

1. 优先取红城近 7 天历史均价中位数。
2. 红城稀疏时，回退多城近 7 天历史均价中位数。
3. 同品质无价时，使用同物品其他品质历史价或当前挂单价，并按 0.85 折扣。
4. 实时市场仍无价时，主手、双手、副手可用本地低价参考库兜底。
5. 不可交易或完全查不到价格的物品显示为无市场价。

当前挂单价只作为兜底。主手、双手、副手如果命中本地低价参考库，且实时挂单兜底价高于参考价 3 倍以上，会按参考价封顶，避免低频装备被单个天价挂单污染补装估值。

补装金额只累加穿戴槽位：

- MainHand
- OffHand
- Head
- Armor
- Shoes
- Bag
- Cape
- Mount
- Potion
- Food

背包 Inventory 不计入补装金额。

## 七、常见问题

### 玩家绑定通过了，但没拿到身份组

检查：

- 机器人是否有「管理身份组」权限。
- 机器人身份组是否高于会员身份组。
- `/设置 会员身份组` 是否配置正确。

### 玩家绑定通过了，但昵称没改

检查：

- 机器人是否有「修改他人昵称」权限。
- 机器人身份组是否高于该成员最高身份组。

### 补装审核员不能审批

检查：

- 是否配置了 `/设置 补装审核身份组 @身份组`。
- 该用户是否已经持有该身份组。
- 点击卡片的人是否在同一个 KOOK 服务器中。

### 补装金额和总损失不一致

这是正常设计：

- 补装金额只算穿戴装备。
- 总损失 = 穿戴装备 + 背包。
- 背包不补，只展示。

### 死亡播报没有马上出现

启动第一轮只记录历史，不会发送旧事件。之后普通时段每 4 分钟轮询一次，20:00-00:30 每 90 秒轮询一次。若仍无播报，检查：

- `/设置 击杀播报频道`、`/设置 阵亡播报频道` 或兜底 `/设置 播报频道` 是否正确。
- bot 是否能在频道发消息。
- `bot.log` 是否有接口或发消息错误。

### 战报没有发到专属频道

检查：

- 是否配置了 `/设置 战报推送频道 #频道`，或兼容命令 `/设置 战报频道 #频道`。
- 本会参战人数是否达到 `/设置 战报本会最小人数 <人数>`，默认 20。
- 当前是否在北京时间 14:30 到次日 05:00 的自动推送窗口。
- bot 是否能在战报频道发卡片消息。
- `bot.log` 是否有 AlbionBB、官方战役详情或 KOOK 发送错误。

### 某些装备无市场价

亚服市场数据稀疏，低频装备可能无成交。当前已有多级兜底：

- 同品质 history。
- 同物品其他品质 history。
- 当前挂单价。
- 本地武器/副手低价参考库。

如果仍然无价，多半是不可交易、数据源缺失，或物品不在武器/副手参考库范围内。

## 八、不建议 AI 自动决定的事项

AI 可以作为辅助说明层，但不建议直接参与核心决策。

可以考虑让 AI 做：

- 给管理员总结补装异常。
- 生成每周战斗报告。
- 把命令说明改写成新手引导。
- 用自然语言查询白名单内的只读状态，例如绑定状态、最近击杀/阵亡、补装进度、补装队列和频道配置概况。

当前已提供：

- `/战报`：基于最近战役生成短摘要。
- `/助手 <问题>`：命令引导和白名单只读查询。普通成员可查本人绑定状态、本人最近击杀/阵亡、本人补装进度；管理员/补装审核员可查补装队列概况；管理员可查频道配置概况。
- `/补装解释 <申请号>`：解释补装金额、背包不计补装、缺价或估值异常。
- 补装审核卡：自动给管理员展示「AI 审核提示」，仅作为只读参考。
- 自动 ZvZ 战报卡：自动附带「AI 摘要」，只总结事实包里已有的参战规模、击杀/阵亡、高光玩家和榜单信息。

不建议让 AI 做：

- 自动批准绑定。
- 自动批准补装。
- 改写补装金额。
- 自动撤销身份组。

这些动作目前应由规则、市场数据、数据库状态和管理员确认共同控制。
